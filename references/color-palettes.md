# Fallback Color Themes

Use these only when the target app does not expose a clear brand palette in its repo. Each fallback theme maps directly to the `IconTheme` token structure from `references/theme-system.md`.

## How to use this file

1. Match the app's mood and category to the closest fallback theme.
2. Copy the full token set instead of mixing rows from different themes.
3. Adjust only if the product context requires it.
4. Keep `mono.fill` as pure white.

## Notes

- Do not merge tokens from different fallback themes unless the repo itself justifies a mixed palette.
- If the icon needs to feel more premium or serious, darken the background slightly before adding more colors.
- If the icon needs more energy, increase the size or contrast of the motif before increasing palette complexity.
